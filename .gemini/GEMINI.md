## UI & Styling Guidelines (Vercel Style)
- **Aesthetics:** The project must follow Vercel's clean, sophisticated design system.
- **Typography:** Use **Inter** for standard text (sans-serif) and a clean monospace font for code or technical details. Do not use overly "artsy" or stylized fonts.
- **Color Palette:** Focus heavily on high-contrast Black and White. The background should be pure white or pure black depending on the theme. Use very subtle shades of gray for borders and backgrounds. Keep colored accents (like primary buttons) extremely minimal, preferring monochrome where possible.
- **Components:** Buttons and cards should have slight border radii (`md` or `lg`, typically 6px or 8px) and a subtle 1px border. Avoid overly soft/rounded designs or aggressive drop shadows unless mimicking an elevated Vercel modal.
- **Naming Conventions:** React components must use `kebab-case` for file names and the default export should follow the same pattern (e.g., `interactive-map.tsx` exporting `InteractiveMap`).
- **Spacing:** Ensure generous whitespace and rigorous alignment to a 4px/8px grid.
